# CI/CD con GitLab

## ¿Qué es GitLab CI/CD?

En términos sencillos, GitLab CI/CD es como un asistente robótico capaz de realizar automáticamente tareas específicas en un ordenador, especialmente aquellas relacionadas con la creación, prueba y despliegue de software. Solo necesitamos darle las instrucciones necesarias.

## ¿Cómo funciona?

El equipo utiliza GitLab CI/CD para automatizar su proceso de desarrollo. Cada vez que se escribe o actualiza una pieza de software, el pipeline ejecuta automáticamente las siguientes acciones:

- 🔧 **Compila el software** para asegurarse de que está correctamente escrito.
- 🧪 **Ejecuta pruebas** para comprobar que todo funciona como se espera.
- 📦 **Crea una versión lista para usar**.
- 🚀 **Despliega la aplicación en un servidor** para que esté disponible para los usuarios.

Todo esto ocurre de forma automática, sin intervención manual.

Mientras la desarrolladora termina su trabajo, el sistema CI/CD trabaja en segundo plano publicando la nueva versión.

## Beneficios

Implementar CI/CD permite:

- ✅ Detectar problemas de forma temprana.
- ✅ Mantener el software siempre en buen estado.
- ✅ Automatizar tareas repetitivas.
- ✅ Reducir errores humanos.
- ✅ Hacer el desarrollo más eficiente.

## Conclusión

El componente CI/CD de GitLab es clave para construir, probar y desplegar aplicaciones de manera automatizada, incluso hasta llevarlas a la nube.
